# Phase 3 — Short Memory

## What landed

- `src/types.ts`: `IndexEntry.kind?: "entity" | "pack"` (optional, backward compat).
- `src/core/registry-scanner.ts`:
  - `manifestEntriesFromManifest` now emits one `{kind: "pack", type: "skill", path: "pack:<name>"}` entry per `packs:` entry, in addition to the existing local-entity entries.
  - `parseIndex` preserves the `kind` field when present in YAML.
- `src/commands/add.ts`:
  - `AddOptions.pack?: boolean | string`.
  - `validateAddFlags`: rejects `--pack` + `--path` / `--local` / `--type` with named-flag errors.
  - `buildDependency`: explicit `--pack` path returns a `PackDependency`; local short-circuit (`tryLocalPackShortcircuit`) returns `{ pack: name }` when `packs.<name>` is defined locally and no source flags are given.
  - `buildPackDep`: factored helper that handles all three pack-ref shapes (local, repo, source-aliased).
  - `checkOverwrite`: detects pack refs on either side and prints "pack reference" wording.
  - `resolveFromRegistries`: when matched IndexEntry has `kind: "pack"`, builds a `PackDependency` instead of an entity dep.
- `src/cli.ts`: `--pack [name]` option (optional value enables `--pack other-name` rename).
- `src/commands/completion.ts`: `--pack` added to bash/zsh completion list.
